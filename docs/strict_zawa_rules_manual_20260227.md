# Strict Zawa 规则与现成员口径（2026-02-27）

## 固定口径
- 排序口径: 一律按 `+2σ` 排序（严格 zawa Monte Carlo），不再使用近似目标函数排序。
- 技能口径: 默认所有卡按满级/进化后技能处理；优先使用工作簿 `UOA大表 新人必看.xlsx` 的“变化后技能”(J列)，否则用主技能(I列)。
- Center口径: 仅 `Véaut / S.teller` 作为队长候选；不枚举其他中心技能卡。
- 候选队口径:
- 先按“目标歌曲颜色感知”的快速目标函数（已包含异色降级与30%同色）为每个队长挑 `TopK` 候选队，再进入 strict zawa 进行 `+2σ` 重排。
- 候选池会额外补入关键同名成员卡与同色高期待值卡，避免 `同名第3/4张关键卡` 被搜索池漏掉。
- 技能倍率口径:
- `S.teller`/`Véaut` 的技能 tuple 按当前歌曲颜色做同色/异色降级（解析规则与 zawa 一致）。
- 技能触发率倍率使用中心吸收后倍率；V/S 吸收时按 `1 + (m-1)*scale` 缩放（如 2.0x 在 0.7 吸收下为 1.7x）。
- 成员名触发率倍率的命中范围（与游戏口径对齐）:
- `S.teller` 作为队长时，成员名倍率仅作用于 `Véaut` 卡；同名普通卡不吃该成员名倍率。
- `Véaut` 作为队长时，同名普通卡可吃成员名倍率。
- 异色降级硬规则（全卡通用）:
- 任何 `S.teller`/`Véaut`（以及同类“同タイプ楽曲のときに効果+X%アップ”卡）在异色歌曲中，**只保留基础值**，不吃 `+X%` 同色追加段。
- 示例（按 tuple 的 D/E 段）:
- `8-16-7-30-0`（同色）在异色降级为 `8-16-7-19-0`（T2）。
- `9-20-9-0-65`（同色）在异色降级为 `9-20-9-0-41`（T2）。
- `7-28-6-19-0`（同色）在异色降级为 `7-28-6-12-0`（T2）。
- `8-38-5-18-0`（同色）在异色降级为 `8-38-5-6-0`（T2）。
- 上述规则不仅限于 `遠藤光莉[S.teller]`、`宮地すみれ[S.teller]`，而是对所有满足该技能结构的卡统一生效（含后续新增异色 Véaut/S.teller）。
- 技能高σ比较口径:
- `3.68%(7-28-6-19)` 不保证在 `+2σ/+3σ` 强于 `3.40%(8-14-9-25)`。
- 在相同倍率（如 `1.7x`）下，`3.40` 常见现象是 `median` 略低但高σ更高（因为持续更长、上限更高，方差更大）。
- 因此最终仍以严格 zawa Monte Carlo 的 `+2σ` 排序为准，不用单一“基础期待值”直接判强弱。
- 卡过滤口径:
- 过滤所有 `期待值/最大期望 < 2.0%` 的卡。
- 过滤所有未填期待值/最大期望的卡（通常为回血、减伤、判定类），因为其 `skill_expected` 视为 0。
- 默认候选池口径（Web优化器）:
- 默认用 `全卡池`（即“现役名单 + T1(期待值>=3.0)`，并保留全部V/S卡）做Top5优化。
- 也可切为“仅持有卡池”做个人账号定制优化。
- 若未手动指定“队长候选”，优化器自动在 `Véaut / S.teller` 队长池搜索。
- V/S技能发动率链路口径:
- `Véaut` 与 `S.teller` 都会吸收前排同 source_type 的技能发动率效果，并按中心链倍率缩放（例: `2.25x` 在 `0.7` 链下生效为 `1.875x`，即 `+87.5%`）。
- `S.teller` 队长下，成员名倍率不会直接作用到同名普通卡；只有 `Véaut` 卡命中该成员名倍率。
- `axis_t1` 候选优先级（默认）:
- 歌曲主色 + 主轴单偏T1 > 歌曲主色 + 主轴双偏T1 > V/S副色 + 主轴单偏T1 > V/S副色 + 主轴双偏T1 > 三偏T1 > 其余。
- 若队长为V卡，自动把V卡副色纳入候选优先级；不排除异色V卡队长。
- 30%类型加成口径:
- 仅当前歌曲同色卡吃到 30%（`ceil(Vo*0.3)+ceil(Da*0.3)+ceil(Pe*0.3)`）。
- 异色卡不吃 30% 类型加成。
- 30%同色加成硬规则（逐卡）:
- 判定条件: `card.color == song.color` 时该卡才有 30%类型加成。
- `card.color != song.color` 时该卡 `Vo/Da/Pe` 的 30%加成全为 `0`。
- 综合力口径:
- 进歌前综合力 = `中心技能生效后(scene Vo+Da+Pe)` + `成员分*5` + `服装Vo/Da/Pe加成` + `家具(office)加成` + `front skin加成` + `skill条目(场景/服装)`.
- 进歌后综合力 = `进歌前综合力` + `30%类型加成总和`。
- 默认“最佳配置”参数（用于组队算分）:
- 服装: 每人 `Vo+125 / Da+125 / Pe+125`。
- 家具(office): `Vo+17% / Da+17% / Pe+17%`（按卡面三围逐卡上取整后求和）。
- front skin: 皮肤档位固定为：
- 单偏：单轴 `+9%`（Vo或Da或Pe）
- 双偏：双轴各 `+8%`（Vo+Da / Da+Pe / Vo+Pe）
- 三偏：三轴各 `+5%`（Vo+Da+Pe）
- `auto` 轴向规则：
- `sum_vo -> max(vo+da, vo+pe)`（各8%，按当前队伍/歌曲色目标下 skin_total 取高）
- `sum_da -> max(vo+da, da+pe)`（各8%，按当前队伍/歌曲色目标下 skin_total 取高）
- `sum_pe -> max(vo+pe, da+pe)`（各8%，按当前队伍/歌曲色目标下 skin_total 取高）
- `sum_vo_da -> vo+da(各8%)`
- `sum_da_pe -> da+pe(各8%)`
- `sum_vo_pe -> vo+pe(各8%)`
- 其他模式（如 `sum_all/max_each`）-> `vo+da+pe(各5%)`
- `auto` 目标类型规则（用于优化器与评估）:
- 若 `front_skin_target_color` 显式指定（如 `R` / `R,G` / `all`），按指定生效。
- 若 `front_skin_target_color=song` 且 `profile=auto`，会在候选目标类型中择优（含歌曲色、队长 source_type、前排颜色单色/双色组合）。
- 因此在 `V` 队长 + 复色前排场景下，可能自动落到 `RED・GREEN` 这类双色皮肤（例如 `Vo、Daが8%アップ`）而非单一歌曲色。
- 示例：队长 `村山美羽 Véaut(sum_vo_da, source=GREEN+RED)`，前排含 `金村美玖 Véaut(GREEN)` 与 `青春の馬 金村美玖(RED)` 时，`auto` 会优先比较并可落到 `R,G + Vo、Daが8%アップ`。
- 兼容旧协议：仍接受 `front_skin_rate/front_skin_axes` 或 `front_skin_vo_rate/da/pe_rate` 覆盖。
- skill条目: 每人 `scene 430 + costume 10`（默认总计 `440*5=2200`）。
- 去重口径:
- Top队伍按“5卡集合”去重；仅队长/队员位置互换但5卡相同，视为同一队，不重复上榜。
- 成员分口径: 默认 `0/人`（由账号成员分主导）。
- 优化器成员分口径:
- 若未提供某成员的成员分（例如新卡/新成员），优化器自动使用 `default_member_point` 回填，并在结果中标记 `default_estimate`，便于后续替换为真实成员分重算。
- 优化器搜索口径:
- UI 不暴露 `每队长候选队数 / shortlist / preselect` 参数，后端使用固定内部参数（当前为 5/50/80/30）保证稳定性与可复现性。
- `preselected` 定义:
- `preselected` = 在候选队集合里，先按 objective 预排序后，进入 strict zawa MonteCarlo 重算的队伍数（即 `preselect_top_m` 截断后的数量）。
- `preselected` 不是总候选数；总候选见 `candidate_count`。
- `preselected` 漏检结论（已实测）:
- 对照脚本: `tools/benchmark_preselect_and_strategy.py`
- 报告: `update/bench_preselect_strategy/20260228_020748/preselect_strategy_benchmark.md`
- 在 fast_all（36候选）下，`pre30` 与 `pre_all` 并非恒等：
- 蓝歌 `東京タワーはどこから見える？` 出现 Top1 差异，`+2σ` 从 `8,719,604`（pre30）提升到 `8,947,360`（pre_all）。
- 其他 4 色在该组参数下无差异。
- 额外两组 seed 复测（`20260228`、`20260301`，trials=200）中，均出现 `2/5` 歌曲 Top1 不一致，说明 `preselected` 截断确实可能漏。
- 结论:
- 若目标是“严格不漏”，应使用 `preselect_all=true` 或显著增大 `preselect_top_m`；代价是耗时明显上升。
- 若目标是“交互速度优先”，保留 `pre30` 但要接受小概率漏掉高σ队伍。
- 候选策略（可选）:
- 新增 `candidate_strategy=axis_t1`（主色+主轴优先，单偏T1→双偏T1→三偏T1；V卡第二色自动纳入）。
- 该策略用于候选排序，不改变 strict zawa 评分器本身。
- 在 `20260228_020748` 的 fast_all 5色基准中，`axis_t1` 与默认策略 Top1 一致，仅耗时略降。
- 全卡池一键优化加速口径:
- 当候选池=全卡池且未手动指定队长时，启用 `fast_all_mode`：先快筛队长（默认前24），内部参数为 `4/30/36`（且可由请求参数上调），并采用两阶段试行（先快后精），用于平衡覆盖率与耗时。
- Web默认严格参数（当前）:
- `candidate_strategy=axis_t1`
- `opt_min_skill_expected=3.0`（T1池，V/S卡保留）
- `preselect_all=true`
- `disable_fast_all=false`（保留快筛队长，但扩大候选覆盖）
- `center_candidates_per_center=16, shortlist=48, search_pool=48`（前端默认）
- 结果仍以严格 zawa MonteCarlo重排输出；meta 中会写明 `internal_search_config` 与 `timing_sec`。
- 同参数请求带内存缓存，重复运行会命中 `cache_hit=true`。
- Group综合力口径: 默认 `1809224`。
- 过滤口径: 仅使用本文件记录的“现成员名单”，不走 wiki 自动刷新。
- 卡面icon口径: 卡池显示优先用 `catalog -> textures/cards/*_icon.png.bytes` 映射到 CDN；映射不到则回退颜色占位。
- 结果展示口径（Web）:
- `Result` 每个 Top 队伍默认显示简表；点击“展开算分细则”后显示：
- `Center Skill/Desc`、`Front tuple明细`、`Scene raw/effective/delta`、`成员分(卡分+成员分)`、`加成合计`。
- 成员分明细中的“卡分”按游戏显示口径使用 `scene card = Vo+Da+Pe+430`（与右侧成员总合口径一致）。
- 异色 `Véaut/S.teller` 在结果里的期望值显示：
- 同时给出 `base期望` 与 `降级后期望`，并显示 `proc倍率` 后的最终期望，避免把异色卡误看成同色3.68。
- 账号配置口径（Web）:
- 支持本地保存多个账号档案（`group_power + member_points`，请求仍带 `default_member_point=0` 兜底）。
- 页面启动若无账号会自动创建 `琴原美名` 并默认加载；若已有 `琴原美名` 也会优先加载。新账号通过弹窗创建并持久化到 `app/data/account_profiles.json`。
