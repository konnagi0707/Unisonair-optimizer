# UOA Scoring UI / 账号配置统一约束（2026-03-01）

## 目标
- 页面以“筛选 + 算分/优化 + 结果”三段为主，优先紧凑、可读、低空白。
- 默认用于“最优配队”，手动单队算分为辅助。
- 所有账号配置（group总综合力、成员分）走后端 JSON 持久化。

## 当前固定规则
- 左栏：卡片筛选（基础筛选 + 成员筛选）。
- 右栏：算分设置 + 成员卡列表（含搜索、重置、持有池操作）。
- 底部全宽：结果（仅展示结果，不再重复放优化入口）。
- 左右两列按 `1:2` 宽度。
- 左右两列主面板底部对齐（同一行高度拉齐）。
- 歌曲下拉按 zawa 顺序（优先 `zawa_index`）显示。
- 高级加成默认按项目规则启用（衣服/家具/skin/同色）。

## 文案与显示
- 删除冗余提示文案（如“成员、颜色、团体、排序都支持筛选”“账号初始设置与歌曲参数”）。
- 成员筛选区域默认全展开，不做内部滚动截断（櫻坂/日向成员区域不内滚）。
- 成员筛选区不保留“清空成员”按钮（统一由顶部“重置筛选”入口处理）。
- 成员卡列表标题显示候选计数；持有池状态单独显示为：
  - `持有卡池: X 张 | 全卡池/仅持有`
- 列表工具栏保留“搜索 + 持有池操作”；“重置筛选”放在算分设置按钮区。

## 刷新保持
- 页面刷新后自动恢复上次操作状态（浏览器本地存储）：
  - 左侧筛选条件（成员/颜色/团体/排序/系列/期望标签/搜索词）；
  - 右侧算分参数（模式/歌曲/group/试行/排序/前N/候选卡池）；
  - 账号选择、成员分当前值、持有/候选/必带、5槽位与左侧折叠状态。
- 自动恢复仅作用于当前浏览器本机，不影响后端账号 JSON 数据。

## 账号配置（后端）
- API:
  - `GET /api/profiles`
  - `POST /api/profiles`
  - `DELETE /api/profiles/{name}`
- 持久化文件：`app/data/account_profiles.json`
- 账号结构：
  - `group_power`
  - `member_points`（成员名 -> 分值）
  - `exclude_codes`（卡码列表，排除池）
  - `saved_at`
- 账号读取交互：
  - 下拉选择账号后不立即覆盖页面；
  - 点击“读取账号”才把该账号的 group/member_points 应用到当前页面。
  - 读取账号时同步应用该账号的 `exclude_codes`。
  - 下拉空值为“默认设置（成员均值15000）”。

## 新建账号弹窗
- 点击“新建账号”打开弹窗。
- 必填：账号名。
- 可填：group总综合力（空值按 `0`，输入框 placeholder 显示 `0`）。
- 成员分：展示全部成员输入框；默认留空；未填按 `0` 保存。
- 保存后：写入后端 JSON，并自动切换到该账号。
- 若账号列表为空，首次加载自动创建并加载 `琴原美名`（使用默认 group/member_points）。
- 若账号列表中存在 `琴原美名`，默认优先加载该账号；否则加载首个账号。

## 成员分行为
- 取消“默认成员分”文本框（只使用账号成员分；无值时按0）。
- 前端默认 `default_member_point=15000`（当未读取账号时作为成员分回退值）。
- 引擎只使用请求中的 `member_points`，缺失成员回落 `default_member_point`（即 0）。
- UI 默认展示：`シーンカード + メンバー = 综合力`。
- 单卡成员分支持“修改成员分 / 使用账号值”切换覆盖。
- 点击“修改成员分”或“使用账号值”时弹出确认框；若即将把成员分改为 `0`，再次确认。
- 点击“保存账号”后，当前页面会立即以最新账号成员分作为基准（`使用账号值` 回退到刚保存的值）。

## 队伍优化入口
- 队伍优化控件固定放在“算分设置”模块：
  - `前N队伍`
  - `候选卡池`
  - `一键重置筛选`
  - `一键全卡池Top5`
  - `最优配队`
  - `optSummary` / `optState` / 进度条
- 约束防呆：
  - `必带` 最多 5 张；
  - 当 `必带=5` 时，必带里必须至少有 1 张可当队长的 V/S 卡，否则无可行队伍；
  - `仅持有卡池` 模式下，如果持有池里没有 V/S 队长卡（Véaut / S.teller），禁止发起优化。
  - 前端 V/S 判定以 `is_vs_base` 为主（tag/title 仅兜底），避免因卡片摘要不含 `vs_rule` 字段导致误判。
- 增加“排除池”用于剔除未持有/不想要的卡：
  - 结果卡片头像旁可直接点 `未持有` 加入排除池（再次点击可取消）。
  - “查看排除”可查询当前排除池，并逐张恢复或“全部恢复”。
  - `清空排除` 可一键清空排除池。
  - 优化请求带 `exclude_card_codes`，后端会从候选池中剔除这些卡。
  - 排除池跟账号走：点“保存账号”后，`exclude_codes` 一起写入账号 JSON。
- `一键重置筛选` 同时清空：左侧筛选条件 + 队长候选 + 必带约束（不清空持有卡池）。
- 结果区域只负责显示 TopN 详细结果与分布表。
- 优化结果主表中直接显示 `Front 编成`（放在“发动效果”下方），不再放进折叠区。
- 折叠细则中移除重复行 `成员分明细(卡分+成员分)`，仅保留底部表格 `成员分明细（卡面分 + 成员分）`。
- 优化执行改为“后台任务 + 前端轮询”：
  - 发起接口：`POST /api/optimize/jobs`
  - 状态接口：`GET /api/optimize/jobs/{job_id}`
  - 刷新页面不打断后端计算；前端会自动恢复轮询并展示结果。

## 顶部参数排版
- `模式 / 选择歌曲 / group总综合力 / 试行回数 / 排序` 同一行展示（宽屏）。
- “选择歌曲”列宽优先，其他输入列压缩。
- 小屏按断点自动换行，不出现横向滚动。
- 账号区按钮（新建/保存/删除）采用紧凑按钮组，不做整行等宽拉伸。
- `一键全卡池Top5` 与 `最优配队` 按钮保持同排、等高、视觉强化（最优配队主CTA）。
- 宽屏下 `账号按钮组` 右对齐、`Top5/最优配队` 与候选设置同一行右侧对齐；中小屏自动换行到两列/单列。
- 算分设置整体采用“双层控制面板”视觉（账号层 + 参数层），通过浅边框与内阴影提升层次，按钮统一圆角/高度/hover反馈。

## 左右主模块对齐
- 左侧“卡片筛选”和右侧“成员卡列表”统一置于同一个外层主面板（single module shell），内部左右分栏。
- 折叠基础筛选/成员筛选后，左栏保留与右栏同底的留白，保持模块底线一致。
- 以左侧主面板实际高度为基准，右侧主面板同步同高（桌面宽度）。
- 右侧成员卡列表采用模块内纵向滚动；左侧成员筛选区不做拉伸填充，避免成员筛选下方空白。
- 左栏折叠状态变化（`details toggle`）后立即触发高度同步，避免出现左右底线短暂错位。

## 弹窗排版
- 新建账号弹窗顶部保留“账号名 + group总综合力”两列。
- 成员分输入区采用多列紧凑网格，单项为“成员名 + 输入框”卡片式行，减少大面积空白。

## 后续排版迭代方向
- 继续压缩模块高度与按钮尺寸，减少空白。
- 保证移动端不出现横向滚动。
- 成员卡片信息按“操作优先 + 技能信息完整”继续微调。
- 卡片筛选左栏保持“分组卡片 + chip 层级 + 轻动效”风格，提升可读性与视觉统一性。
