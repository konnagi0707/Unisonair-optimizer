# 队伍优化模拟

面向 UNI'S ON AIR 的本地 Web 队伍优化与算分工具。

本项目专注两件事:

- 快速管理“我的账号配置”（group 总综合力、成员分、持有卡池）
- 在给定歌曲和约束下，稳定得到可复现的 Top 队伍结果，并支持换卡横向对比

## 1. 核心功能

- 卡池筛选
  - 颜色 / 团体 / 排序 / 系列 / 期望标签 / 成员筛选
- 账号配置
  - 下拉自动加载账号 / 自动保存开关 / 账号操作（新建、保存、账号备份、删除）
  - 账号持久化到 `app/data/account_profiles.json`
- 单队算分
  - 精确输出分布、综合力拆分、技能明细、成员分明细
- 精确配队（TopN）
  - 支持候选卡池、队长候选、必带约束
  - 后台任务执行，页面刷新可恢复轮询
- 持有卡池管理
  - 在卡列表/结果/换卡对比中直接切换“已持有/未持有”
- 队伍换卡对比
  - 在 Top 结果中点击成员进入换卡弹窗
  - 替换后可和原队伍横向比较核心指标

## 5. 首次使用（推荐流程）

1. 打开页面，在右侧“账号配置”下拉框选择账号（选中即自动加载）。
2. 检查“自动保存：开/关”状态；建议保持开启。
3. 在“选择歌曲”里选目标曲。
4. 选择“候选卡池”（通常先用“仅持有卡池”）。
5. 点击“精确配队”获取 TopN（或用“一键配队”走快捷流程）。
6. 在结果中点击成员卡，进入“队伍换卡对比”做替换验证。

## 6. 页面说明

## 6.1 左侧: 卡片筛选

- 基础筛选: 颜色、团体、排序、系列、期望标签
- 成员筛选: 按团体/期别选择成员

说明:

- “3.68%(S)”与普通“3.68”已区分，不会混淆。
- “毕业成员”表示未分期成员组。

## 6.2 右侧上半: 算分设置

- 账号区:
  - 账号下拉: 选中后自动加载到页面
  - 自动保存: 控制是否在配置变化时自动写回当前账号
  - 账号操作: 新建账号 / 保存账号 / 账号备份 / 删除账号
  - 新建账号: 弹窗录入 group + 全成员分
  - 保存账号: 将当前页面配置覆盖写回账号
  - 删除账号: 删除当前账号
- 参数区:
  - 模式 / 选择歌曲 / group 总综合力 / 试行回数 / 排序
- 优化操作:
  - 前N队伍
  - 候选卡池
  - 一键重置筛选
  - 一键配队（快捷：全卡池 + Top5）
  - 精确配队（按当前候选设置完整优化）
  - 取消优化

## 6.3 右侧下半: 成员卡列表

每张卡支持:

- 上队
- 持有/未持有切换
- 候选（仅 V/S 卡可设）
- 必带
- 成员分手动修改/恢复账号值

## 6.4 底部: 结果

- 展示 Top1~TopN 队伍
- 每队显示:
  - 5 张卡摘要（可点击进入换卡）
  - 进歌前/后综合力
  - 分布表（MIN~MAX）
  - 发动效果
  - Front 编成
- 展开细则后可查看:
  - Center Skill
  - Scene raw/effective/delta
  - 加成合计
  - 成员分明细（含 Vo/Da/Pe、卡面分、成员分）

## 7. 账号与数据持久化

## 7.1 后端账号文件

- 文件: `app/data/account_profiles.json`
- 每个账号包含:
  - `group_power`
  - `member_points`
  - `owned_codes`
  - `exclude_codes`（预留字段）
  - `saved_at`

## 7.2 浏览器本地状态

本地会保存:

- 筛选条件
- 算分参数
- 当前槽位
- 最近结果
- 进行中的优化任务 ID

因此刷新后通常会恢复到上次状态。

## 8. 算分口径（简版）

- 卡分: `Vo + Da + Pe + 430`
- 进歌前综合力:
  - `center生效后scene + 成员分 + 衣装 + 家具 + skin + skill条目`
- 进歌后综合力:
  - `进歌前 + 同色类型30%`
- 最终分布:
  - 由 zawa Monte Carlo 计算得到 `min ~ max` 与各 sigma 档

完整推导与参数说明:

- `docs/spec_coding_uoa_scoring_app.md`
- `docs/strict_zawa_rules_manual_20260227.md`

## 9. 队伍换卡对比怎么用

1. 在结果区点击任意队员卡。
2. 弹窗中先点“正在替换第 N 位”的槽位。
3. 右侧筛选候选卡（支持成员、系列、持有状态）。
4. 点击“替换为此卡”或“与第X位互换”。
5. 点击“计算替换对比”。
6. 查看原始 vs 替换后的差值（综合力、sigma、分布）。

## 10. 常见问题

### Q1: 为什么结果里会出现“未持有”卡?
- 你当前候选卡池是“全卡池”时会出现。
- 改成“仅持有卡池”即可只在持有池内搜索。

### Q2: 为什么提示没有可行队伍?
常见原因:

- 必带=5 且没有 V/S 队长卡
- 仅持有模式下持有池没有 V/S 队长卡
- 约束太硬，候选组合为空

### Q3: 为什么有些卡没有 icon?
- 可能该卡资源在本地 catalog 中不存在精确映射。
- 详见 `reports/icon_diagnosis_20260301.md`。

### Q4: 为什么和游戏里有个位数差异?
- 上取整链路、skin/家具配置、期望锚点、歌曲色判定可能导致小偏差。
- 建议先核对参数完全一致再比较。
